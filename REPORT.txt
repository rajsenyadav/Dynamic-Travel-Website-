🌍 Avipro Travels - Travel Package Booking System
A modern Content Management System (CMS) for managing and booking travel packages online. Built with HTML , CSS , PHP, MySQL, and JavaScript.
________________


✨ Features
For Users
* Browse travel packages with detailed information
* Search and filter by destination, date, and price
* Submit booking requests with real-time validation
* Contact form with AJAX submission
* Fully responsive design (mobile, tablet, desktop)
For Administrators
* Secure admin panel with login system
* Add, edit, delete travel packages
* View and manage customer bookings
* Update site content and settings
* Dashboard with statistics
Security
* Password hashing (Bcrypt)
* SQL injection prevention
* XSS protection
* Session-based authentication
* Client and server-side validation
________________


🛠️ Technologies
Backend: PHP 7.4+, MySQL 5.7+
Frontend: HTML5, CSS3, JavaScript (AJAX)
Server: Apache
Tools: XAMPP, FileZilla, phpMyAdmin
________________


🚀 Quick Setup
1. Install XAMPP
Download from apachefriends.org and start Apache + MySQL
2. Create Database
-- In phpMyAdmin (http://localhost/phpmyadmin)
CREATE DATABASE avipro_travels;
-- Import database.sql file


3. Configure
Edit config/config.php:
define('DB_HOST', 'localhost');
define('DB_USER', 'root');
define('DB_PASS', '');
define('DB_NAME', 'avipro_travels');
define('SITE_URL', 'http://localhost/avipro-travels/');


4. Access
* Website: http://localhost/avipro-travels/
* Admin: http://localhost/avipro-travels/admin/
* Login: admin / admin123

* bookings - Customer bookings
* site_settings - Site configuration
* contact_messages - Contact inquiries
* package_images - Package gallery

🐛 Troubleshooting
CSS not loading?
* Clear cache (Ctrl+F5)
* Check SITE_URL in config.php ends with /
Database connection error?
* Verify credentials in config.php
* Check MySQL is running
* Ensure database exists
Images not showing?
* Check image paths in database
* Use placeholder images (see code)
